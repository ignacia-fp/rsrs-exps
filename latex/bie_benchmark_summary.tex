\section*{BIEGrid Benchmark Summary ($k=p=40$)}

Square BIEGrid, double precision, $\kappa=0$, one right-hand side,
GMRES tolerance $10^{-10}$, \texttt{num\_threads=4}, and
\texttt{save\_samples=false}, \texttt{load\_samples=false}.
Reported quantities:
\[
\text{relerr}=\frac{\|A-A_{\mathrm{rsrs}}\|_2}{\|A\|_2},
\qquad
\text{errsolve}=\|I-A_{\mathrm{rsrs}}^{-1}A\|_2.
\]
The $\mathrm{cond}(A)$ column uses the PerLevel estimate.

\section*{Sampling-Budget Update}

For the serial runs below we use square BIEGrid in double precision with
$\kappa=0$ and one right-hand side. We set \texttt{num\_threads=1}, use
deterministic seed 12345, and keep
\texttt{save\_samples=false}, \texttt{load\_samples=false}.

The fixed-rank budget update is written in terms of the user inputs
$k$ (fixed rank) and $p$ (oversampling parameter):
\[
s_{\mathrm{tot}}=\max(s_{\mathrm{vec},k})+p.
\]
In the implementation this total is rounded to a nearby multiple of 5; that detail
is not important for the present comparison.
PerLevel uses this fixed total budget $s_{\mathrm{tot}}$ for sampling while keeping
level-dependent active counts. Constant uses $(s_{\mathrm{tot}}, s_{\mathrm{tot}})$ at every level.
